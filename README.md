# Cate Chip

`Cate Chip` 是一个 **“结构优先”计算系统软硬件协同设计** 的研究型项目。

项目当前聚焦第一阶段：以有限维向量空间中的双积结构为入口，建立一条从数学结构、程序表示、编译优化到周期级执行验证的闭环路径。换句话说，它要验证的不只是“局部计算能否做快”，而是“对象如何拆分、拼接、投影、注入、复制、汇总与重组”能否成为一等的系统约束，并最终落到可运行的编译器与执行模型中。

---

项目的核心目标是通过工程项目实施逐步沉淀出一种全新的计算系统软硬件协同设计方法。

短期目标是，开发一个向量计算芯片验证方法的可行性（具备双积结构，对称幺半闭结构）

长期目标是，开发一个软硬件协同工作的高阶推理芯片（具备笛卡尔闭结构）

---

当前仓库包含两类核心内容：

- `docs/`：数学基础、项目方法论、执行模型和技术规格文档
- `csimulator/`：第一阶段 Rust 实现的周期级软件模拟器

## 项目目录

当前根目录保持尽量简洁，主要包含以下内容：

- `docs/`
   项目的正式文档目录，包含当前有效版本的宣言、数学说明、执行模型、技术规格、模拟器选型说明和微架构接口规格。

- `docs/deprecated/`
  已归档的历史版本和废弃文档，保留演化过程，便于回溯设计思路，但不应作为当前实现依据。

- `csimulator/`
  第一阶段 Rust 原型工程。这里实现了 `BlockMat IR -> optimize -> lower -> schedule -> simulate -> oracle -> report` 的完整链路，并提供 benchmark、JSON artifact 导出和可视化脚本。

## 当前状态

仓库当前已经完成第一阶段原型的主要闭环：

- 文档层：核心问题、数学结构、执行模型、技术规格已形成当前版本
- 实现层：`csimulator` 已能对双积结构对应的块矩阵态射进行优化、调度、周期级模拟与 oracle 对照
- 实验层：已内置从小规模到 128 维混合结构的 benchmark，并支持导出 JSON artifact 与可视化结果

如果你的目标是快速进入实验，建议直接进入 [csimulator/README.md](csimulator/README.md) 查看运行方式；如果你的目标是理解设计出发点和规范边界，建议优先阅读 `docs/` 下的文档。
